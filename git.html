Einführung in GIT

Bei der Softwareentwicklung gibt es keinen Weg an einer Versionsverwaltung vorbei. Dabei gibt es unterdessen einige Alternativen – GIT ist eine davon. In diesem Artikel möchte ich einen Überblick und einen einfachen Einstieg in die Welt von GIT ermöglichen. Ich werde nicht auf alle Konzepte detailliert eingehen, weil dies definitiv die Länge eines Blogeintrags sprengen würde. Ich verweise jedoch gerne auf die offizielle GIT-Homepage, welche eine sehr ausführliche Dokumentation besitzt, am Anfang jedoch etwas überfordern kann.

Warum GIT?

Ein grosser Unterschied von GIT gegenüber anderen Versionsverwaltungen (z.B SVN) ist die Weise, wie Dateien und deren Änderungen betrachtet werden. Während SVN sich eine Liste von fortlaufenden Änderungen merkt (“Diffs”), speichert GIT sich sogenannte Snapshots. Ein Snapshot stellt den momentanen Zustand sämtlicher Dateien dar, bei dem jedoch alle unveränderten Dateien nur als Referenz gemerkt werden. Dadurch ergeben sich einige Vorteile.

GIT ist schnell

Da sich mit GIT immer eine lokale Kopie des Repositories auf dem Rechner befindet sind alle Operation sehr schnell. Es können auch ohne Netzwerkzugriff Änderungen vorgenommen und commitet werden. Es ist keine Verbindung zu einem Server nötig. Erst wenn die lokalen Änderungen auf den Server übertragen (“gepusht”) werden sollen muss eine Verbindung bestehen.

Branchen ist einfach und schnell

Mit GIT kann innerhalb von Sekunden ein lokaler Branch erstellt werden. Dort kann ein spezifisches Feature weiterentwickelt werden und der Entwickler entscheidet später selber, ob er die Änderungen anderen zur Verfügung stellen möchte oder wieder verwirft.

Verteilung von lokalen Kopien

Da jeder Entwickler eine lokale Kopie des gesamten Repositories besitzt gibt es auch automatisch viele Backups. Jeder Entwickler könnte jederzeit den Stand auf dem Hauptserver mit seinem Stand ersetzen.

Staging Area

In diesem Bereich können modifizierte Dateien vor dem Commit nochmals angesehen werden. Es ist möglich trotz anderen offenen Änderungen einzelne Dateien zu commiten.
